# Deploying DClaw Flow (Vercel + Render + Neon)

Free-tier, no-credit-card stack:

| Piece | Host | Notes |
|-------|------|-------|
| Postgres | **Neon** | Free, persistent |
| Backend (FastAPI) | **Render** | Free Docker web service; spins down after ~15 min idle |
| Frontend (Next.js) | **Vercel** | Free Hobby |

Order matters because of a CORS/URL chicken-and-egg: **DB → backend → frontend → back-fill CORS**.

> The repo is deploy-ready: `render.yaml` (Blueprint), and `backend/Dockerfile`
> runs `alembic upgrade head` on boot so the schema **and indexes** (migrations
> 002/003, which `create_all` does NOT create) exist in production.

## 1. Neon (Postgres)

1. Create a free project at neon.tech → database name `dclaw_flow`.
2. Copy the connection string and convert it for SQLAlchemy + **asyncpg**:
   - scheme `postgresql://` → `postgresql+asyncpg://`
   - **remove** `sslmode=...` and `channel_binding=...` query params (asyncpg rejects them)
   - **add** `?ssl=require`

   ```
   postgresql+asyncpg://USER:PASSWORD@ep-xxx.neon.tech/dclaw_flow?ssl=require
   ```

   This is your `DATABASE_URL`.

## 2. Render (backend)

1. New → **Blueprint** → connect this repo. Render reads `render.yaml` and
   creates the `dclaw-flow-api` web service (Docker, free plan, `rootDir: backend`).
2. Set the env vars left as `sync: false`:
   - `DATABASE_URL` = the Neon asyncpg URL from step 1
   - `CORS_ORIGINS` = a placeholder for now (e.g. `http://localhost:3003`) — back-filled in step 4
   - `OPENROUTER_API_KEY` = optional (leave blank to keep the heuristic copilot)
   - `WEBHOOK_SECRET` / `ADMIN_TOKEN` are generated by Render automatically.
3. Deploy. On boot the container runs `alembic upgrade head` then uvicorn on
   `$PORT`. Verify `https://<service>.onrender.com/health` → `{"status":"ok"}`.
   Note the service URL (e.g. `https://dclaw-flow-api.onrender.com`).

## 3. Vercel (frontend)

1. New Project → import this repo → set **Root Directory = `frontend`**
   (Vercel auto-detects Next.js).
2. Env var `NEXT_PUBLIC_API_URL` = the Render URL from step 2
   (e.g. `https://dclaw-flow-api.onrender.com`). It is baked at build time.
3. Deploy. Note the frontend URL (e.g. `https://dclaw-flow.vercel.app`).

## 4. Back-fill CORS

On Render, set `CORS_ORIGINS` = the Vercel URL from step 3 and redeploy
(or save — Render redeploys on env change). The browser can now call the API.

## Smoke test

- Open the Vercel URL → the app loads, the Copilot widget answers.
- Create a workflow, run it, check Execution History.
- (Webhook) Activate a webhook workflow, `POST` raw JSON to
  `https://<render-url>/api/v1/flows/webhooks/<path>` → `202`.

## Notes / gotchas

- **Cold starts:** Render free spins down after ~15 min idle; the first request
  then takes ~30–60s. Fine for demos.
- **Render free budget:** 750 instance-hours/month per workspace — enough for one
  mostly-idle backend, not several always-on ones.
- **Retention cron:** see `execution-retention.md` — point it at the Render URL.
- **Frontend `Dockerfile`** is for local `docker-compose` only; Vercel builds
  Next.js natively and ignores it.
